Reliable UDP Transport Implementation

Project: Plivo Systems Challenge Objective: Lossless delivery under strict latency constraints

Problem Statement

The assignment required developing a transport protocol capable of handling reliable data delivery over a lossy, hostile relay. The core challenge was satisfying a strict deadline constraint while ensuring total bandwidth overhead remained under 2.00x.

Architectural Overview

I designed a multi-threaded, non-blocking architecture that decouples packet I/O from the playout engine. This ensures that network jitter does not stall the delivery pipeline. The system operates on a hybrid recovery strategy:

  • Fractional FEC (80%): By XORing the current payload with the previous one and attaching it to 80% of packets, I enabled instant recovery for single-packet drops without waiting for retransmission.
  • Gap-Based ARQ: For burst-loss events exceeding parity capability, the receiver actively scans for sequence gaps and issues "Triple-Tap" NACKs, ensuring feedback loop resilience.

System Level Design

The implementation relies on an event-driven model:

  • Concurrency Control: Mutex-protected buffers allow the playout_loop to safely read data while the ingress_thread writes packets, preventing race conditions.
  • Rate Limiting: A strict 20ms cooldown on NACK requests prevents "NACK storms" and keeps the system within the bandwidth budget during high-loss scenarios.
